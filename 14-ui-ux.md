# 14 — UI/UX 设计

## 14.1 设计总则

**核心体验目标**：让用户感觉自己打开了一个与虚拟人物的聊天窗口，而不是一个调试 AI 的控制面板。

**视觉风格**：简洁、有质感、不粉红、不二次元（除非用户自定义主题）。默认风格应该是中性的、有温度的、克制的。

### 状态栏设计

```
顶部显示角色当前状态，但不机械：
✅ "林晓 · 正在上课" （简洁）
✅ 角色忙时显示小图标 🕐
✅ 如果角色在睡觉，对话页底部提示"林晓已经睡了，明天早上会看到你的消息"
❌ "角色状态：unavailable，可用性：0.2" （太技术化）
❌ 不要显示"AI 正在生成回复..."
```

### 延迟回复的 UI 表现

```
用户发送消息后：

如果角色 normal（正常回复）：
- 消息显示"已送达"
- 几秒到几分钟后显示回复
- 不需要"对方正在输入"（不是即时通讯，不要滥用）

如果角色 busy（忙碌中）：
- 消息显示"已送达"
- 状态栏提示"林晓正在上课，可能会晚点回"
- 不要假装正在输入

如果角色 sleeping（睡觉）：
- 消息显示"已送达"
- 底部提示"林晓已经睡了"
- 第二天角色醒来后会"看到"并回复
```

### 已送达 / 已读

```
✅ 显示"已送达"（消息发送成功）
✅ 可选显示"已读"（角色回复时默认已读）
❌ 不要显示精确的"对方正在输入……"
❌ 不要显示"对方已读但未回"（制造焦虑）
```

### 记忆编辑的权限设计

```
可编辑：
✅ 内容（修正不准确的记忆）
✅ 重要性（标记为重要/不重要）
✅ 关联（手动关联其他记忆）
✅ 删除

不可编辑：
❌ 类型（系统决定）
❌ 来源消息（不可篡改证据链）
❌ 置信度（系统计算，除非用户手动纠正触发重新计算）
```
